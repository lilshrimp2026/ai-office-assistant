## 🏗️ 组件关系说明

### 1. 前端层 (Vercel)
- **技术栈**: Next.js 14 + React + Tailwind CSS
- **职责**: 
  - 用户界面渲染
  - 客户端状态管理
  - API 请求封装
- **免费额度**: 100GB 带宽/月，足够中小项目

### 2. API 网关层 (Cloudflare Workers)
- **技术栈**: Hono (轻量级框架)
- **职责**:
  - 统一 API 入口
  - JWT 鉴权验证
  - 请求限流 (防滥用)
  - AI 服务路由 (根据任务复杂度选择 Groq 或 Gemini)
  - 请求日志记录
- **免费额度**: 10 万次请求/天

### 3. 数据层 (Supabase)
- **技术栈**: PostgreSQL + Supabase Auth
- **职责**:
  - 用户数据存储
  - 文档/对话历史记录
  - 订阅状态管理
  - 行级安全策略 (RLS)
- **免费额度**: 500MB 数据库 + 1GB 文件存储

### 4. 存储层 (Cloudflare R2)
- **技术栈**: S3 兼容存储
- **职责**:
  - 用户上传的文件存储
  - AI 生成的文档存储
  - 图片/附件管理
- **免费额度**: 10GB 存储 + 1000 万次读取/月

### 5. AI 服务层
- **Groq**: 快速推理，适合简单问答、文本处理
  - 免费额度：1400 次请求/天
  - 模型：Llama 3.1 70B
- **Google Gemini**: 复杂任务，适合多模态、长文本
  - 免费额度：1500 次请求/分钟
  - 模型：Gemini 1.5 Flash

## 💰 0 成本启动策略

| 服务 | 免费额度 | 预计支撑用户数 |
|------|---------|--------------|
| Vercel | 100GB 带宽/月 | ~1000 DAU |
| Cloudflare Workers | 10 万请求/天 | ~500 DAU |
| Supabase | 500MB DB + 1GB 存储 | ~1000 用户 |
| Cloudflare R2 | 10GB 存储 | ~500 用户 (平均 20MB/用户) |
| Groq | 1400 请求/天 | ~200 DAU (7 次/用户) |
| Gemini | 1500 请求/分钟 | 充足 |

**扩展策略**: 
- 用户增长后，优先升级 Groq 配额 ($0.07/1000 tokens)
- 存储增长后，升级 R2 ($0.015/GB)
- 数据库增长后，升级 Supabase ($25/月 起)

## 🔐 安全设计

1. **认证**: Supabase Auth (JWT + Refresh Token)
2. **授权**: 行级安全策略 (RLS) + API 网关鉴权
3. **限流**: Cloudflare Workers 中间件 (每用户 100 请求/小时)
4. **加密**: 敏感数据 AES-256 加密存储
5. **CORS**: 严格域名白名单

## 📈 扩展性设计

1. **会员订阅**: 预留 `subscriptions` 表，支持 Stripe 集成
2. **支付集成**: API 设计预留 webhook 接口
3. **多租户**: 数据库设计支持团队/企业版
4. **插件系统**: AI 服务抽象层，可接入更多模型
